3 approcci per costruire Veleggiamo.com

Scegli la direzione — poi definiamo il design dettagliato

A

Sito editoriale con link affiliati manuali

Come funziona: Scrivi articoli su barche, coste, natura. In ogni articolo inserisci link affiliati a Viator/GetYourGuide copiati manualmente dal loro portale affiliati.

Tech: Next.js SSG + MDX per gli articoli. Zero API esterne.

Pro

  • Lanci in 2 settimane
  • Zero dipendenze API
  • Massima semplicità

Contro

  • Catalogo gite statico
  • Prezzi/disponibilità non aggiornati
  • Scalabilità limitata
B ⭐ Consigliato

Content Hub + Catalogo gite via API affiliati

Come funziona: Articoli editoriali scritti da te (o con AI) + catalogo gite aggiornato automaticamente tramite Viator API e GetYourGuide API. Le gite mostrate sono sempre live, con prezzi reali.

Tech: Next.js ISR + Viator Partner API + GYG Partner API + Supabase per cache + schema.org per Google.

Pro

  • Catalogo sempre aggiornato
  • Rich snippet su Google (stelle, prezzi)
  • Scala senza lavoro manuale
  • Riutilizza il codebase esistente

Contro

  • Richiede approvazione API Viator/GYG (1-2 sett.)
  • 2-4 settimane per il lancio
C

Content Hub + API + Newsletter + Community

Come funziona: Come B, ma aggiunge una newsletter settimanale ("La gita del weekend"), pagine di recensioni utenti, e possibilità per operatori locali di segnalare le loro gite (non prenotabili sul sito, ma linkati).

Tech: Come B + Resend/Mailchimp per email + tabella reviews su Supabase.

Pro

  • Massimo engagement e ritorno utenti
  • Community fidelizzata = SEO boost (dwell time)
  • Newsletter = canale diretto senza dipendere da Google

Contro

  • 4-6 settimane di sviluppo
  • Newsletter richiede contenuto costante
Nota sul codebase esistente: Il progetto Next.js + Supabase che hai già costruito è una base ottima. Va riprogettato nelle pagine (non più fornitori/prodotti) ma l'infrastruttura — auth, DB, API routes — si riutilizza quasi tutta.
